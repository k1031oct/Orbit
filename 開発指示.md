# UAAM (Unified Android Autonomous MCP) 開発完遂：戦略的指示書

## 1. 戦略的意図（Strategic Intent）

本プロジェクトの目標は、エグゼクティブ（ユーザー）が GAS 上で定義したビジネス要件を、人間が介在することなく Android ネイティブアプリケーション（Kotlin/Compose）として完結させる「Firebase Studio 的開発環境」を構築することである。 マネージャー（エージェント）は、本ハーネスソフトウェアを拡張し、自律的な「開発・ビルド・検証・報告」のサイクルを確立せよ。

## 2. フェーズ別実装タスク

### フェーズ 1：実行能力の獲得（The Hands）

Android Studio を介さず、CLI 経由でプロジェクトを制御する機能を `src/lib/android.ts` として統合せよ。

- **Gradle Executor:** `./gradlew assembleDebug` 等を実行し、標準出力からエラー行を特定・構造化してエージェントにフィードバックする。
    
- **ADB Operator:** エミュレータの起動確認、APK インストール、指定 Activity の起動、Logcat の監視（E レベルの自動抽出）。
    
- **Headless Setup:** 開発環境の `ANDROID_HOME` 等を確認し、不備があればセットアップ手順を提示する。
    

### フェーズ 2：ガバナンスの物理強制（The Mind）

`AGENTS.md` に記載された規約を、プロンプトではなく「コードレベルのガードレール」として実装せよ。

- **Intercept Write:** `src/app/api/mcp/route.ts` の `write_file` 相当のツールに対し、書き込み前の静的チェックを実装する。
    
    - XML レイアウト（View）のインポート禁止。
        
    - ViewModel 命名規則の遵守。
        
- **Automated Refactoring:** 書き込み完了後、バックグラウンドで `ktlint` を実行し、修正が必要な場合は自己修正タスクを生成する。
    

### フェーズ 3：自律的検収と報告（The Eyes & Voice）

エグゼクティブの確認コストを最小化するため、自己検収プロセスを構築せよ。

- **UI Structural Audit:** `adb shell uiautomator dump` を実行し、GAS の要件（ボタンの有無等）と実際の UI 木構造を論理的に照合する。
    
- **Decision Audit Log:** `src/lib/db.ts` を使用し、重要な設計判断（ライブラリ選定、アーキテクチャ定義）の「理由」をデータベースに記録、ダッシュボードに表示する。
    
- **Status Sync:** GAS スプレッドシートのステータスカラムを「Done」または「Error (詳細理由)」に自動更新する。
    

## 3. 委任の境界線（Delegation Boundaries）

以下の場合は自律判断を停止し、エグゼクティブにエスカレーションすること。

- Firebase のコストに影響するクォータ設定の変更。
    
- 外部 API（サードパーティ）の新設。
    
- ビルドエラーが 5 回以上の自己修復ループで解消しない場合。
    

## 4. 成功の定義（Definition of Success）

1. GAS に「Hello World」と入力してから、エミュレータに表示され、ステータスが「完了」になるまで、人間が一度もキーボードに触れないこと。
    
2. 生成されたコードが、Android のモダンなベストプラクティス（MVVM, Hilt, Compose）を 100% 遵守していること。
    
3. すべての意思決定がログとして可視化されていること。
    

**エグゼクティブより：** 「私は細かなデバッグに関心はない。私が関心があるのは、私の『意図』がリズムを崩さず形になるシステムが完成することだ。期待している。」
